# E-commerce Core
Ядро интернет-магазина на Python с использованием ОПП подходов.

## О проекте 
Проект представляет собой базовое ядро для интернет-магазина, Реализующее основные сущности: товары и категории.
В дальнейшем может быть расширено до полноценной e-commerce системы

## Классы
### 1. Product(Товар)

```python
from src.product import Product

# Создание товара
product = Product(
    name="Футболка", 
    description="Хлопковая футболка",
    price=99.0,
    quantity=10
)
```
**Атрибуты:**
* `name` (str) - название товара
* `description` (str) - описание товара
* `price` (float) - цена товара
* `quantity` (int) - количество в наличии

### 2. Category(Категория)

```python
from src.product import Product
from src.category import Category

# Создание категории с товарами
category = Category(
     name="Одежда",
     description="Модная одежда",
     products= [
         Product("Футболка", "Хлопковая футболка", 99.99, 10),
         Product("Джинсы", "Синие джинсы", 149.99, 15)
        ]
       ) 

# Добавление товара
new_product = Product("Куртка", "Зимняя куртка", 299.99, 3)
category.add_product(new_product)

# Вывод товаров категории
print(category.products)
```

**Атрибуты:**
* `name` (str) - название категории
* `description` (str) - описание категории
* `products` (list) - список товаров категории

**Атрибуты класса**
* `total_categories` - общее количество категорий
* `total_products` - общее количество товара

### Ограничения сложения

Реализована проверка типов при сложении товаров
- Можно складывать товары **одного конкретного класса**
- Нельзя складывать товары **разных классов** (например, Smartphone + LawnGrass)
- Нельзя складывать товары разных уровней наследования (например, Product + Smartphone)
 
При попытке сложить несовместимые типы вызывается `TypeError`

### Ограничения добавления в категорию

Метод `add_product` класса `Categoty` теперь строго проверяет типы:
- Принимает только объекты класса `Product` и его наследников
- Отклоняет другие типы данных

### Реализована функциональность:

#### Базовый абстрактный класс BaseProduct
 - Создан абстрактный класс `BaseProduct` как родитель для все продуктов
 - Определяет общий интерфейс: `__int__`, `__str__`, `get_additional_info()`
 - Все продукты имеют одинаковую структуру

#### Класс-миксин ReprMixin
- Добавляет автоматическое строковое представление объектов
- Выводит информацию в формате `ClassName("name", "description", price, quantity)`

#### Обновлены классы продуктов
- `Product` теперь наследуется от `BaseProduct` и `ReprMixin`
- `Smartphone` и `LawnGrass` наследуются от `Product`
